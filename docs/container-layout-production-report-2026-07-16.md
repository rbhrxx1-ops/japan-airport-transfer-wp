## 五、缓存根因与处置

发布后命令行连续抓取 `style.css?ver=0.3.2` 均返回新哈希，但真实 Chromium 曾命中 Hostinger CDN 的旧首页 HTML，旧 HTML 仍引用 `style.css?ver=0.3.0`，造成 1440px 与 1280px 同一轮验收结果不一致。浏览器资源诊断确认问题来自边缘 HTML 缓存，而不是生产文件漂移或代码回退。

通过已安装 LiteSpeed 插件的公开 `litespeed_purge_all` 动作执行官方缓存清理后，首页首次响应即为 `HCDN=MISS`，并引用 `style.css?ver=0.3.2`。随后五视口生产布局门禁全部稳定通过。

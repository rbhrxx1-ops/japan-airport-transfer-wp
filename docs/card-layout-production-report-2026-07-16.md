## 二、根因与修复范围

`.jat-card` 同时使用 `height: 100%`、padding 和 border，但原规则仍采用浏览器默认的 `content-box`。Grid 轨道计算的是内容高度，padding 与 border 被额外加在轨道之外，导致卡片真实外框超出所在行，并覆盖下一行或后续区块。

| 项目 | 处理结果 |
|---|---|
| 根因修复 | 为 `.jat-card` 增加 `box-sizing: border-box` |
| 缓存刷新 | 主题版本从 `0.3.0` 更新为 `0.3.1` |
| 生产业务内容 | 未修改 |
| 日语文案、价格、公司及法务资料 | 未修改 |
| 预约插件、数据库结构与订单状态机 | 未修改 |
| 品牌 Logo 与图标 | 未修改 |
| 固定高度、负边距、隐藏文字 | 未使用 |

经真实几何采集确认，现有首页最大宽度、四列/两列/单列断点、服务页两列/单列规则在盒模型修正后已经恢复正常，不需要继续扩大卡片、压缩文字或改变全站容器宽度。这样可以用最少代码解决问题，并避免对其他页面产生连锁影响。

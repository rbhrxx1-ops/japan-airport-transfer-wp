# 2026-07-16 首页与服务页卡片布局修复

## 范围

根据中村先生提供的生产截图，修复首页第二屏服务卡片及 `/service/` 第一屏服务卡片因盒模型计算错误造成的卡片跨行覆盖、露字与后续内容遮挡问题。未修改日语文案、价格、公司/法务内容、品牌资产、预约逻辑或数据库结构。

## 根因与实现

`.jat-card` 原有 `height: 100%` 采用默认 `content-box`，padding 与 border 被加在 Grid 轨道高度之外，导致卡片外框超出网格行。以最小规则 `.jat-card { box-sizing: border-box; }` 修复，并将主题版本更新为 `0.3.1` 以刷新浏览器缓存。

## 测试工具

新增 `tests/card_layout_e2e.py`，覆盖首页和服务页 1440、1280、1024、768、390px 共十个生产场景，检查卡片相交、网格包裹、后续区块间距、响应式列数及横向溢出。反向变异验证确认移除修复时门禁会失败。

生产主机会拦截默认自动化指纹，因此将 `tests/recruit_page_e2e.py` 与 `tests/sticky_header_e2e.py` 对齐已经验证的正常匿名浏览器 User-Agent；固定页头测试同时增加 HTTP 200 前置检查。测试修正不涉及生产业务逻辑。

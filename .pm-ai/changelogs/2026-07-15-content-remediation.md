## 一、任务与事实边界

本轮依据中村先生提供的《机场_车站迎接服务（MeetandGreet）竞品与现状分析整改报告》执行。外部报告只作为问题线索；公开事实以生产只读核验、项目 `truth.md` 和中村先生已确认内容为准。整改前形成 `docs/content-remediation-matrix-2026-07-15.md`，将建议分为立即实施、部分实施、已具备或误报、等待业务确认四类。

未经确认的具体价格、WhatsApp 或 LINE 渠道、固定响应时限、全天候承诺、覆盖全部地区、实际客户案例、图片素材、语言能力和服务人员资质均未发布。报告所称“特商法重复”和“电话号码不一致”经生产前台与 WordPress 数据核验不成立，因此未修改现有法务正文。

## 三、备份、部署与回滚

生产写入前已通过 UpdraftPlus 原生接口创建并核验新的完整备份，备份集包含数据库、插件、主题、上传和其他内容。同时在生产账户下创建权限为 `0700/0600` 的专用回滚目录、内容种子文件快照、目标页面部署前清单与 MariaDB 原生数据库导出。完整路径、大小、哈希和恢复说明记录在生产报告中。

生产仅临时上传经过本地测试的 `bin/seed-site.php`，校验远端哈希与 PHP 语法后幂等执行一次。随后使用 WordPress 与 LiteSpeed 原生入口清理页面缓存。最终公网安全复核发现该脱离 WordPress 上下文的脚本 URL 返回 HTTP 500；再次核对三类主机指纹及本地/远端 SHA-256 一致后，已删除远端一次性脚本和空 `bin` 目录，复测 URL 返回 HTTP 404。仓库继续保留版本化脚本；插件、主题、订单、预约表、用户、媒体和已发布公司/法务正文均未修改。

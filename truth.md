# truth.md — Japan Airport Transfer WordPress 项目唯一事实源

**最后更新：** 2026-07-11（GMT+9）  
**维护角色：** ②全栈工程师AI（生产部署）  
**汇报对象：** PM-AI（通过中村先生转达）

## 1. 项目目标

在不破坏现有生产站、全过程可追踪、可验收、可回滚的前提下，为 `japanairporttransfer.com` 建设日语 WordPress 举牌迎送服务网站，并开发五步在线预约程序。生产发布必须先经过测试环境验证、PM-AI 验收以及中村先生转达的上线授权。

首期标准服务为：`空港お迎え（ミート）`、`空港お見送り（センディング）`、`駅お迎え`、`駅お見送り`。`乗継サポート` 与 `アテンドサービス` 首期仅提供日语咨询入口。

> 在线提交仅代表 `受付済み`，不代表 `予約確定`。价格、资源、集合点和服务可用性由客服人工确认。

## 2. 身份与权限边界

| 项目 | 事实 |
|---|---|
| 当前角色 | ②全栈工程师AI（生产部署） |
| 可做 | 修改 WordPress 主题/插件、数据库迁移、构建与测试、Git、测试环境、经授权的生产部署、追加 `.pm-ai/changelogs/` |
| 不可做 | 自行改变需求方向；无任务书修改生产；跳过 PM-AI 验收；编造价格、公司资料或服务承诺；复制竞品内容/素材 |
| 汇报链 | 中村先生 → PM-AI → 中村先生转达给②；②完成后向 PM-AI 回报并等待验收 |
| 防越权 | 每次修改文件前复核本节；超范围立即停止并报告 |

## 3. 已批准技术决策

| 维度 | 决策 |
|---|---|
| CMS | WordPress |
| 主题 | 专属主题 `jat-meet-theme` |
| 业务插件 | 专属插件 `jat-reservation` |
| 编辑方式 | 原生区块编辑器、受控区块与结构化字段；不使用重型页面构建器 |
| 前端语言 | 100% 自然日语；仅品牌名、URL、机场代码、航班号及客户输入可保留必要外文 |
| 核心地点 | 羽田空港、成田空港、東京駅、品川駅；其他地点为咨询入口 |
| 预约流程 | 五步：服务与日期 → 旅客信息 → 迎送内容 → 申请人/车辆 → 确认与同意 |
| 数据 | 专属结构化数据表、版本化迁移、不可顺序推测的受付编号、状态历史、邮件日志与审计 |
| 状态机 | `受付済み → 確認中 → お見積り送付済み → 予約確定 → サービス完了`，支持 `変更確認中`、`キャンセル` |
| 首期不做 | 自动报价、在线支付、航班 API、自动派员、客户中心、实时地图、CRM 同步、Logo 上传 |
| 发布策略 | 测试环境先行；生产发布前完整备份并验证回滚；不在生产后台临时编辑源代码 |

## 8. 质量与安全要求

| 类别 | 必须满足 |
|---|---|
| 表单 | 服务器端白名单、类型/长度校验、输出转义、预处理查询、Nonce、蜜罐/反垃圾、速率限制、幂等键、重复提交检测 |
| 可恢复性 | 网络错误、弱网、Nonce 过期不得清空用户输入；重复请求不得生成重复订单 |
| 隐私 | 最小采集、角色权限、默认遮蔽、审计、保留期限、导出/删除支持、备份治理 |
| 无障碍 | 标签、错误摘要、焦点、键盘、对比度、减少动画；错误不只依赖颜色 |
| 日语 | 页面、弹窗、日期、浏览器校验、Cookie、404、权限、邮件均不得出现中文或非必要英文 UI |
| SEO | 唯一标题/描述/H1、canonical、站点地图、robots；测试站禁止索引；只为可见内容输出结构化数据 |
| 版权 | 原创日语文案；仅用自有或授权素材；不公开 PDF 未授权地图或旅客图像 |
